class Sensor¶

This is the top-level class used by the SDK. In nominal operation, the user will instantiate a sensor object and interact directly with it. Consequently, public Sensor methods are all direct API.

Sending Commands

enum class SendCommandBlockMode¶

Block mode for sending a command.

See also

sendCommand()

Values:

enumerator None¶

Do not wait for a response from the unit before returning.

enumerator Block¶

Block upon a response from the unit until the specific timeout; will return ResponseTimeout if timeout is hit.

enumerator BlockWithRetry¶

Block upon a response from the unit until the specific timeout; retry sending the command and blocking commandSendRetriesAllowed times before returning ResponseTimeout.

Error writeSettings() noexcept¶

Sends a Write Settings command to the unit and blocks on the unit’s confirmation.

Error reset() noexcept¶

Sends a Reset command to the unit and blocks on the unit’s confirmation. After confirmation, it sleeps for resetSleepDuration and verifies sensor connectivity, calling autoConnect if failed.

Error restoreFactorySettings() noexcept¶

Sends a Restore Factory Settings command to the unit, blocks on the unit’s confirmation, reopens serial at the unit’s default baud rate, sleeps for resetSleepDuration, then verifies unit connectivity. If THREADING_ENABLE, resets the Listening Thread.

Serial Connectivity

using BaudRate = Registers::System::BaudRate::BaudRates¶
Error connect(const Serial_Base::PortName &portName, const BaudRate baudRate) noexcept¶

Opens the serial port at the specified baud rate. This does not verify any connectivity, and as such only validates that the serial port is available for opening. If THREADING_ENABLE, this starts the Listening Thread.

Parameters

  • portName – The name of the port to connect.

  • baudRate – The baud rate at which to connect.

Error autoConnect(const Serial_Base::PortName &portName) noexcept¶

Opens the serial port, scanning all possible baud rates until the unit is verified to be connected. This performs a verifySensorConnectivity at each possible baud rate. If THREADING_ENABLE, this starts the Listening Thread.

Parameters

portName – The port name to which to connect.

bool verifySensorConnectivity() noexcept¶

Sends a ReadRegister for the Model register. Returns true if a valid response is received, otherwise returns false.

inline std::optional<Serial_Base::PortName> connectedPortName() const noexcept¶

Gets the port name of the open serial port. If no port is open, will return std::nullopt.

inline std::optional<BaudRate> connectedBaudRate() const noexcept¶

Gets the baud rate at which the serial port is opened. If no port is open, will return std::nullopt.

Error changeBaudRate(const BaudRate newBaudRate) noexcept¶

Sends a Write Register for the new baud rate to the unit and reopens to the serial port under the new baud rate. Will retry on failure.

Error changeHostBaudRate(const BaudRate newBaudRate) noexcept¶

Changes the host (computer) baud rate without commanding a change to the VectorNav unit’s baud rate. Use with caution.

void disconnect() noexcept¶

Disconnects from the unit. If THREADING_ENABLE, this closes the Listening Thread.
